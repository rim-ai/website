2024 National Data Science Competition by RIMAI: Focusing on License Plate Recognition in Mauritania

Join us in a pioneering computer vision challenge hosted by RIMAI, designed to revolutionize license plate recognition in Mauritania. We invite innovators, data scientists, and AI enthusiasts to develop algorithms that enhance accuracy and efficiency. This competition aims to create one of the first AI-based systems capable of recognizing Mauritanian license plates. The challenge comprises two parts: the first, 'Data Collection' (a data engineering challenge), focuses on gathering diverse and quality images, while the second, 'Data Science' (in a Kaggle competition form) involves creating and tuning the recognition algorithms.

The first part (Data Collection) starts on May 18th and ends on June 1st 2024.

The second part (Data Science) starts on June 4th and ends on June 18th 2024.

I) Data Collection Part:

Task

For the first part, participants are tasked with compiling a unique dataset of Mauritanian license plates, crucial for the competition's second phase. Image quality is paramount as it directly affects the robustness and accuracy of the computer vision models. Data collection should be hybrid: 60% through web scraping and 40% from original photographs.

Submission

Deadlines:

Format & Conditions:
  • Participants can join the competition either individually or in a team (of max 5 people).
  • A zip (named "data_collect_group_id") should be composed of only images following the description defined before:
    • Web scrapped images: these images should be named in the format "web1_{image_name}.[jpeg, png, gif]" (60%).

  • Each submission should not exceed the limit of 2GB.
  • You are allowed an unlimited number of submissions per team (or individual).
  • The code used for web scraping should be shared through a public GitHub repository. If you don't have a GitHub account, create one and share the link with us upon submission of your code when you push it.

Scoring Method:

Participants will receive scores based on the quality, number, and uniqueness of the images they upload. The quality score for each image is computed using the following metrics:

  • Sharpness Score: Calculated using the Laplacian variance of the image, normalized between 0 and 1.
  • Color Vibrancy Score: Computed from the standard deviation of the saturation channel in the HSV color space, normalized by dividing by 128.0.
  • Contrast Score: Determined by the difference between the minimum and maximum luminance values, normalized by their sum.
  • Noise Score: Based on the standard deviation of the grayscale image, normalized and inverted so that lower noise results in a higher score.
